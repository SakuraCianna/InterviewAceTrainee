## 目录结构说明

```
InterviewAceTrainee/
├── .env                  ← Docker Compose 配置（镜像标签、连接池大小等 17 个字段）
├── docker-compose.yml    ← 服务编排文件
├── docker/               ← PostgreSQL / Redis / RabbitMQ 配置镜像源码
├── data/                 ← 三类服务的本机持久化数据（不要提交、移动或直接编辑）
│   ├── postgres/         ← PostgreSQL 18 数据
│   ├── redis/            ← Redis AOF/RDB
│   └── rabbitmq/         ← RabbitMQ 节点数据
├── nginx/                ← Nginx 配置 + SSL 证书
├── secrets/              ← 所有密码和 API 密钥（17 个文件，权限 0700，不要改权限）
├── Backend/.env          ← 旧 Python 版本的配置备份，Spring Boot 不读取，忽略即可
├── Frontend/.env         ← 前端构建时用，服务器上只是存档，忽略即可
└── scripts/              ← Windows 本地开发用的 PowerShell 检查脚本，服务器上用不到
```

**一句话总结：**
- 日常改配置 → 改根目录 `.env`
- 改密码 / API Key → 改 `secrets/` 里对应的文件，然后重启受影响的服务
- `docker/` 的配置在构建时进入镜像，不再需要服务器 `runtime-config/`
- `Backend/.env`、`Frontend/.env`、`scripts/` 都是开发工具，不会挂载进生产容器

## 快速查看状态

```bash
cd /home/ubuntu/InterviewAceTrainee

# 查看所有容器状态

# 查看某个服务的实时日志

# 查看最近 100 行日志

```

## 一、首次启动（全新部署）

按顺序执行，每步等容器 healthy 后再继续：

```bash
cd /home/ubuntu/InterviewAceTrainee

# 1. 显式创建数据目录并设置三个镜像运行用户的权限
install -d -m 750 data data/postgres data/redis data/rabbitmq
docker run --rm --user 0:0 --entrypoint sh \
  --mount type=bind,source="$PWD/data/postgres",target=/target \
  pgvector/pgvector:0.8.5-pg18-bookworm@sha256:12a379b47ad65289572ea0756efc11b7c241a6662833e8af7038cd3b73d647e0 \
  -c 'chown 999:999 /target; chmod 0750 /target'
docker run --rm --user 0:0 --entrypoint sh \
  --mount type=bind,source="$PWD/data/redis",target=/target \
  redis:8.8.0-alpine@sha256:9d317178eceac8454a2284a9e6df2466b93c745529947f0cd42a0fa9609d7005 \
  -c 'chown 999:1000 /target; chmod 0750 /target'
docker run --rm --user 0:0 --entrypoint sh \
  --mount type=bind,source="$PWD/data/rabbitmq",target=/target \
  rabbitmq:4.3.2-management@sha256:c1e33461287f4c53049ce525575f0df77b2a00a88e2593776afbb6c1489bdb81 \
  -c 'chown 999:999 /target; chmod 0750 /target'
docker compose --env-file .env config --quiet

# 2. 启动基础设施（约 30 秒变 healthy）
docker compose --env-file .env up -d postgres redis rabbitmq

# 确认三个都是 healthy 再继续
docker compose --env-file .env ps

# 3. 执行数据库迁移（首次必须跑，跑完容器自动退出）
docker compose --env-file .env run --rm --no-deps migrate

# 4. 启动业务服务
docker compose --env-file .env up -d material-parser api frontend

# 5. 等 api 变 healthy，再启动 Worker 和 Nginx
docker compose --env-file .env up -d worker nginx
```

## 二、日常重启（代码无 schema 变更）

配置改了、密码改了，或者服务异常时用这个：

```bash
cd /home/ubuntu/InterviewAceTrainee

# 重启全部服务
docker compose --env-file .env restart

# 只重启某一个服务
docker compose --env-file .env restart api
docker compose --env-file .env restart worker
docker compose --env-file .env restart nginx
```

改了 `.env` 或 `secrets/` 里的文件后，需要让容器重新读取配置：

```bash
# 重新创建并启动（数据不会丢失）
docker compose --env-file .env up -d --force-recreate api worker
```

## 三、更新代码后重启（带数据库迁移）

从 GitHub 拉取新版本代码后执行：

```bash
cd /home/ubuntu/InterviewAceTrainee

# 1. 拉取最新代码
git pull

# 2. 停止业务服务（保留数据库 / 缓存 / 消息队列，数据不丢失）
docker compose --env-file .env stop nginx worker api material-parser frontend

# 3. 执行数据库迁移（不确定有没有变更时跑一次也无害）
docker compose --env-file .env run --rm --no-deps migrate

# 4. 重新启动
docker compose --env-file .env up -d material-parser api frontend worker nginx
```

## 五、数据库备份

```bash
cd /home/ubuntu/InterviewAceTrainee

# 备份（文件名自动带时间戳）
docker compose --env-file .env exec postgres   pg_dump -U mianba_owner mianba_prod   > "backup-$(date +%Y%m%d-%H%M%S).sql"

# 确认备份文件不为空
ls -lh backup-*.sql | tail -1
```

恢复备份（覆盖现有数据，谨慎操作）：

```bash
# 先停止 API 和 Worker
docker compose --env-file .env stop api worker

# 恢复（把 backup.sql 换成实际文件名）
cat backup.sql | docker compose --env-file .env exec -T postgres   psql -U mianba_owner mianba_prod

# 重启
docker compose --env-file .env start api worker
```

## 六、更新密码或 API Key

```bash
# 例如更新 DeepSeek API Key
echo -n "新的key值" > /home/ubuntu/InterviewAceTrainee/secrets/deepseek_api_key
chmod 444 /home/ubuntu/InterviewAceTrainee/secrets/deepseek_api_key

# 重启读取该密钥的服务（Worker 用 DeepSeek）
docker compose --env-file .env up -d --force-recreate worker
```

各密钥对应的服务：

| 密钥文件 | 需要重启的服务 |
|---|---|
| deepseek_api_key | worker |
| postgres_*_password | postgres + migrate + api + worker |
| redis_app_password | redis + api |
| rabbitmq_*_password + rabbitmq-definitions.json | rabbitmq + api + worker |
| jwt_secret | api |
| resend_api_key、mail_from | api |
| tencent_* | api |
| hcaptcha-* | api |
| material-parser-token | material-parser + api |

## 七、常见问题排查

```bash
# 某个容器一直重启 / 启动失败
docker compose --env-file .env logs --tail=50 <服务名>

# 查看所有容器健康状态
docker compose --env-file .env ps

# 查看资源占用
docker stats --no-stream

# 磁盘和内存
df -h && free -h

# 数据存在哪里（只查看目录元数据，不直接修改内容）
find data -maxdepth 1 -type d -printf '%p\n'
# data/postgres  ← 数据库
# data/redis     ← Redis 缓存与向量索引
# data/rabbitmq  ← 消息队列
```
