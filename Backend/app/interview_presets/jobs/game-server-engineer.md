# 游戏服务端工程师

## 资料来源
- 官方/可靠底座：人社部计算机程序设计员标准；分布式系统、数据库、消息队列和游戏服务架构实践。
- 面试优先级：MMO、卡牌、竞技、房间制、实时战斗或社交系统优先。

## 面试关注点
- 是否能讲清房间/匹配、状态同步、战斗校验、排行榜、背包、经济系统和反作弊。
- 是否理解高并发、数据一致性、延迟、断线重连和灰度发布。
- 是否能说明一次线上事故和回滚方案。

## 追问规则
- 优先追问：战斗结果如何校验、并发峰值怎么扛、道具重复发放如何避免、作弊如何发现。
- 对“做过后端”要追问玩法业务理解。

## 评分维度
- 高分：服务架构、玩法理解、一致性、安全和排障都能讲。
- 扣分：只会普通 CRUD，不理解游戏业务。
